What goes where

MainActivity.kt

App startup only. Keep this file small.

WardrobeApp.kt

App-level navigation and screen wiring.

navigation

Routes and route helpers.

ui/screens

One screen per file.

ui/components

Reusable UI pieces shared across screens.

ui/utils

Constants, helpers, and preference helpers.

Maintenance rules

  • Keep one screen per file whenever possible.
  • Move reusable UI into ui/components.
  • Put constants and utility functions in ui/utils.
  • Do not let MainActivity.kt grow again.
  • Build after each split when refactoring.
  • Remove private from top-level items that must be used in other files.

Suggested splitting order

Safer order for future refactors.

1

Navigation + utils first

Split routes, constants, and welcome preferences.

2

Common components

Move shared UI widgets before moving screens.

3

Dialogs and pickers

Keep modal UI together for easier maintenance.

4

Cards

Move repeated card layouts into the components folder.

5

Screens one by one

Move each screen separately and test after each move.
